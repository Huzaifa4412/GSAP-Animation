What is GSAP?

GreenSock Animation Platform — the gold standard of web animation

Overview

GSAP (GreenSock Animation Platform) is a JavaScript library that lets you animate anything JavaScript can touch — DOM elements, CSS properties, SVG attributes, canvas objects, WebGL, React state, Three.js objects — with blazing speed and rock-solid cross-browser compatibility.

Why GSAP over CSS?

  • Fine-grained playback control (pause, reverse, seek)
  • Sequence complex multi-element animations easily
  • Animate any numeric property, not just CSS ones
  • Superior performance with transforms batching
  • Precise timing and synchronisation via Timelines
  • Rich ecosystem of plugins

Core Concepts

  • Tween — a single animation from A to B
  • Timeline — a container that sequences tweens
  • Easing — controls the rate of change
  • Plugin — extends GSAP capabilities
  • Ticker — GSAP's internal render loop
GSAP is framework-agnostic. It works with vanilla JS, React, Vue, Angular, Svelte, Next.js — anywhere JavaScript runs.

Three ways to add GSAP to your project

Option A — CDN (fastest start)

HTML

<!-- Optional plugins (add only what you need) -->

Option B — npm / yarn

BASH
npm install gsap
# or
yarn add gsap
JavaScript / ESM
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

Always register plugins! Plugins must be registered with gsap.registerPlugin() before use, or they won't work correctly — especially after tree-shaking.

gsap.to / from / fromTo

The three fundamental tween methods — the building blocks of all GSAP animations

gsap.to() — Animate TO a destination

Animates an element FROM its current state TO the values you specify. This is the most commonly used method.

JavaScript
gsap.to(".box", {
  x: 300,        // translateX 300px
  rotation: 360, // rotate 360 degrees
  duration: 1.5, // takes 1.5 seconds
  ease: "power2.out"
});
gsap.to() — Live Demo
BOX

gsap.from() — Animate FROM a starting state

Starts the animation FROM the values you specify and animates TO the element's natural/current state. Great for entrance animations.

JavaScript
gsap.from(".box", {
  opacity: 0,
  y: -60,
  scale: 0.5,
  duration: 1,
  ease: "back.out(1.7)"
});
gsap.from() — Entrance Animation
IN

gsap.fromTo() — Full control: explicit start and end

You specify BOTH the starting values and ending values explicitly. Most precise method — avoids relying on the element's current state.

JavaScript
gsap.fromTo(".box",
  // FROM (starting values)
  { x: -200, opacity: 0, scale: 0.5 },
  // TO (ending values)
  { x: 200, opacity: 1, scale: 1, duration: 1.2, ease: "power3.out" }
);
gsap.fromTo() — Explicit control
→

Targets — What you can animate

  • ".class" — CSS selector (all matching elements)
  • "#id" — ID selector
  • element — Direct DOM node reference
  • [el1, el2] — Array of elements
  • NodeList — Result of querySelectorAll
  • object — Any plain JS object with numeric properties

Everything you can animate and how to express values

Always prefer transform-based properties — they are composited on the GPU and don't cause layout reflow.

Special Value Formats

JavaScript
// Relative values (add/subtract from current)
gsap.to(".box", { x: "+=100" });   // move 100px more to the right
gsap.to(".box", { rotation: "-=45" }); // rotate 45° counter-clockwise

// String with unit
gsap.to(".box", { width: "50vw", fontSize: "2rem" });

// Random value
gsap.to(".box", { x: "random(-200, 200)" });
gsap.to(".box", { x: gsap.utils.random(-200, 200) });

// Snapping to increments
gsap.to(".box", { x: 300, snap: { x: 50 } }); // snaps to nearest 50px

// Multiple targets - function-based values (unique per element)
gsap.to(".box", {
  x: (index, element) => index * 100,  // each box gets different x
  duration: 1
});

Easing

The soul of animation — how motion accelerates and decelerates

What is easing?

Easing defines the rate of change over time. Linear = robotic. Ease curves = natural, organic motion. GSAP has the most comprehensive easing system of any animation library.

Format: "family.direction(strength)" e.g. "power3.inOut" or "elastic.out(1, 0.3)"

Ease FamilyDirectionsCharacter
power0–power4in, out, inOutPolynomial curves — power4 is very dramatic
backin, out, inOutSlight overshoot — like anticipation
elasticin, out, inOutSpringy bounce with oscillation
bouncein, out, inOutBall-bounce effect (no overshoot)
sinein, out, inOutGentle sine wave curve
circin, out, inOutCircular arc — very sharp finish
expoin, out, inOutExponential — extreme acceleration
linear—Constant speed — robotic but useful for loops
steps(n)—Jump to N discrete steps (sprite animation)
none—Same as linear
Click an ease to see it
▶
JavaScript — Ease Examples
// Basic ease
gsap.to(".box", { x: 300, duration: 1, ease: "power2.out" });

// Parametric eases — pass arguments
gsap.to(".box", { y: -100, ease: "elastic.out(1, 0.4)" });
// elastic.out(amplitude, period)
// back.out(overshoot)  — default overshoot is 1.70158

// Ease for specific property only
gsap.to(".box", {
  x: 300,
  rotation: 360,
  ease: "power2.out",       // applied to all
  // or use keyframes for per-property easing
});

// Reversed directions
// "in"    → starts slow, ends fast (ease into motion)
// "out"   → starts fast, ends slow (ease out of motion)  
// "inOut" → slow start, fast middle, slow end

Callbacks

Hook into animation lifecycle events

All Tween Callbacks

CallbackFires WhenReceives
onStartAnimation begins playing—
onUpdateEvery render frame during playback—
onCompleteAnimation finishes forward—
onReverseCompleteAnimation finishes in reverse—
onRepeatEach time a repeat cycle starts—
onStartParamsArguments to pass to onStartarray
onCompleteParamsArguments to pass to onCompletearray
JavaScript
const tween = gsap.to(".box", {
  x: 400,
  duration: 2,
  repeat: 1,
  yoyo: true,

  onStart: () => console.log("🎬 Animation started!"),
  
  onUpdate: function() {
    // 'this' refers to the tween — access progress
    console.log("Progress:", this.progress().toFixed(2));
  },
  
  onComplete: () => console.log("✅ Animation done!"),
  
  onRepeat: () => console.log("🔁 Repeating..."),
  
  onReverseComplete: () => console.log("⏮ Reversed to start"),
});

// Callbacks with params
gsap.to(".box", {
  x: 200,
  onCompleteParams: ["#myElement", 42],
  onComplete: (el, num) => {
    console.log("Done!", el, num);
  }
});

Controls & Playback

Pause, reverse, seek, speed — GSAP gives you a VCR for animations

Tween & Timeline Special Properties

PropertyType / ValuesDescription
durationnumber (seconds)How long the animation takes
delaynumber (seconds)Wait before starting
repeatnumber (-1 = infinite)How many times to repeat
yoyobooleanReverse direction on each repeat
repeatDelaynumberPause between repeats
pausedbooleanStart in paused state
reversedbooleanStart reversed
timeScalenumberSpeed multiplier (0.5 = half speed, 2 = double)
overwriteboolean/"auto"Kill conflicting tweens

Playback Methods

JavaScript
const tween = gsap.to(".box", {
  x: 400, duration: 3, repeat: -1, yoyo: true, paused: true
});

// Playback controls
tween.play();           // start / resume
tween.pause();          // pause at current position
tween.resume();         // resume from paused position
tween.reverse();        // play backwards
tween.restart();        // jump to start and play

// Seeking
tween.seek(1.5);        // jump to 1.5s mark
tween.progress(0.5);    // jump to 50% through animation

// Speed
tween.timeScale(2);     // double speed
tween.timeScale(0.5);   // half speed

// Read state
tween.time();           // current time in seconds
tween.progress();       // current progress 0–1
tween.duration();       // total duration
tween.isActive();       // currently playing?
tween.paused();         // is paused?

// Kill / clean up
tween.kill();           // destroy the tween entirely

// Kill all tweens on an element
gsap.killTweensOf(".box");
gsap.killTweensOf(".box", "x,opacity"); // kill only specific properties

Timelines

Sequence, overlap, and orchestrate complex animations with precision

Why Timelines?

Without timelines, you'd chain animations using delay values — fragile and hard to maintain. Timelines let you sequence animations, adjust timing globally, and control the whole sequence as one unit.

JavaScript — Timeline Basics
// Create a timeline
const tl = gsap.timeline({
  repeat: -1,
  yoyo: true,
  defaults: { duration: 0.5, ease: "power2.out" }
});

// Add tweens — they play IN SEQUENCE by default
tl.to(".box1", { x: 200 })
  .to(".box2", { y: 100 })     // starts after box1 finishes
  .to(".box3", { rotation: 360 }); // starts after box2 finishes

// ─── Position Parameter ───────────────────────────────

// "<" — start at same time as previous tween begins
tl.to(".box", { x: 200 }, "<");

// "<0.2" — 0.2s after previous tween STARTS
tl.to(".box", { x: 200 }, "<0.2");

// "+=" / "-=" — relative to END of timeline
tl.to(".box", { x: 200 }, "+=0.5"); // 0.5s after last tween ends
tl.to(".box", { x: 200 }, "-=0.3"); // overlap: 0.3s before last tween ends

// Absolute time in seconds
tl.to(".box", { x: 200 }, 1.5); // at exactly 1.5s mark

// Labels
tl.addLabel("phase2");
tl.to(".box", { opacity: 0 }, "phase2");
tl.to(".box", { scale: 2 }, "phase2+=0.3");

Timeline nesting

Timelines can be nested inside other timelines. This is powerful for building modular, reusable animation components.

JavaScript — Nested Timelines
// Reusable animation function
function entranceAnim(element) {
  const tl = gsap.timeline();
  tl.from(element, { opacity: 0, y: 30, duration: 0.5 })
    .from(element.querySelector("h2"), { opacity: 0, y: 15 }, "-=0.2");
  return tl;
}

// Master timeline
const master = gsap.timeline();
master.add(entranceAnim("#header"), 0)
      .add(entranceAnim("#nav"), 0.3)
      .add(entranceAnim("#hero"), 0.6);

// Control the whole show with master.pause() / .play() etc.

Stagger

Animate multiple elements with cascading delays — one of GSAP's most powerful features

Basic Stagger

When you target multiple elements, stagger adds an incremental delay between each element's animation. This creates beautiful cascading effects with minimal code.

JavaScript
// Simple stagger — 0.1s between each
gsap.from(".item", {
  opacity: 0, y: 30,
  duration: 0.6,
  stagger: 0.1,  // 0.1s delay between each element
  ease: "power2.out"
});

// Advanced stagger object
gsap.from(".grid-item", {
  opacity: 0, scale: 0.5,
  duration: 0.8,
  stagger: {
    amount: 1.5,          // total stagger time spread across all elements
    from: "center",       // start from center, spread outward
    // from: "end"        // reverse order
    // from: "random"     // random order
    // from: [2, 3]       // start from specific grid position
    grid: [4, 8],         // declare grid dimensions for 2D stagger
    axis: "y",            // stagger along Y axis only
    ease: "power2.inOut"  // ease the stagger timing itself!
  }
});

Keyframes

Define multi-step animations within a single tween — like CSS keyframes but more powerful

JavaScript
// Array-based keyframes
gsap.to(".box", {
  keyframes: [
    { x: 0,   y: 0,    scale: 1,   duration: 0 },
    { x: 200, y: 0,    scale: 1.2, duration: 0.5 },
    { x: 200, y: 150,  scale: 1,   duration: 0.5 },
    { x: 0,   y: 150,  scale: 0.8, duration: 0.5 },
    { x: 0,   y: 0,    scale: 1,   duration: 0.5 }
  ]
});

// Percentage-based keyframes (like CSS @keyframes!)
gsap.to(".box", {
  keyframes: {
    "0%":   { x: 0,    y: 0 },
    "25%":  { x: 200,  y: 0 },
    "50%":  { x: 200,  y: 100 },
    "75%":  { x: 0,    y: 100 },
    "100%": { x: 0,    y: 0 },
    easeEach: "sine.inOut"  // ease between each keyframe
  },
  duration: 2,
  ease: "none"
});

gsap.set() & defaults()

Set properties instantly and configure default values for clean, DRY code

JavaScript
// ── gsap.set() — instant property change (no animation)
gsap.set(".box", { x: 0, opacity: 0, scale: 0.8 });
// Equivalent to duration: 0 — useful for initial states

// ── gsap.defaults() — global defaults for ALL tweens
gsap.defaults({
  duration: 0.8,
  ease: "power2.out"
});
// Now ALL tweens use these defaults unless overridden

// ── Timeline defaults — scoped to that timeline
const tl = gsap.timeline({
  defaults: { duration: 0.5, ease: "back.out(1.7)" }
});
tl.to(".a", { x: 100 })      // uses timeline defaults
  .to(".b", { y: 50 })       // uses timeline defaults
  .to(".c", { rotation: 180, duration: 1 }); // overrides duration

// ── gsap.getProperty() — read computed GSAP value
const xPos = gsap.getProperty(".box", "x");
const opacity = gsap.getProperty(".box", "opacity");

// ── gsap.utils — useful utility functions
gsap.utils.toArray(".item");              // convert NodeList to Array
gsap.utils.clamp(0, 100, value);         // clamp a value
gsap.utils.mapRange(0, 1, 0, 100, 0.5); // map 0.5 → 50
gsap.utils.wrap(0, 5, 6);               // wrap index → 1
gsap.utils.random(10, 50);              // random number

ScrollTrigger Plugin

Tie animations to scroll position — the most powerful scroll animation library available

Setup

ScrollTrigger must be registered after importing.

JavaScript
gsap.registerPlugin(ScrollTrigger);

// ── Basic scroll-triggered animation
gsap.from(".section", {
  opacity: 0,
  y: 60,
  duration: 1,
  scrollTrigger: {
    trigger: ".section",    // element that triggers the animation
    start: "top 80%",       // "trigger-edge viewport-edge"
    end: "bottom 20%",      // when to end
    toggleActions: "play none none reverse",
    // toggleActions: "onEnter onLeave onEnterBack onLeaveBack"
    // options: "play", "pause", "resume", "reset", "restart", "complete", "reverse", "none"
  }
});

// ── Scrubbing — animation tied to scroll position
gsap.to(".box", {
  x: 500, rotation: 360,
  scrollTrigger: {
    trigger: ".box",
    start: "top center",
    end: "+=500",           // 500px of scrolling
    scrub: true,            // link animation to scroll
    scrub: 1,               // 1 = 1 second lag for smooth feel
    pin: true,              // pin the element during animation
    pinSpacing: true,       // add space to prevent content jump
    markers: true,          // show debug markers (remove in production!)
  }
});

// ── Snap scrolling
ScrollTrigger.create({
  trigger: ".container",
  snap: {
    snapTo: [0, 0.25, 0.5, 0.75, 1], // snap to these progress values
    duration: { min: 0.2, max: 0.5 },
    ease: "power1.inOut"
  }
});

// ── Batch — efficient stagger on scroll for many elements
ScrollTrigger.batch(".card", {
  onEnter: batch => gsap.from(batch, {
    opacity: 0, y: 30, stagger: 0.1
  })
});

// ── Horizontal scroll
gsap.to(".panels", {
  xPercent: -100 * (panels.length - 1),
  ease: "none",
  scrollTrigger: {
    trigger: ".container",
    pin: true,
    scrub: 1,
    snap: 1 / (panels.length - 1),
  }
});

// Basic draggable

Create your own easing curves with CustomEase

JavaScript
// CustomEase uses SVG path syntax for the curve
// M 0,0 means "start at 0,0" (beginning of animation)

  y: -200,

GSAP Ticker

Hook into GSAP's optimized render loop for frame-perfect custom logic

What is the Ticker?

GSAP manages its own requestAnimationFrame loop — the ticker. Instead of creating your own rAF loop, you can hook into GSAP's — it's synchronized with all other animations and is more efficient.

JavaScript
// Add a function to the ticker
gsap.ticker.add((time, deltaTime, frame) => {
  // time       — total elapsed time in seconds
  // deltaTime  — milliseconds since last frame
  // frame      — frame count
  
  myCanvas.clear();
  myCanvas.draw(time);
});

// Remove a listener
const myFn = () => { ... };
gsap.ticker.add(myFn);
gsap.ticker.remove(myFn);

// FPS capping (default is uncapped, matches display refresh)
gsap.ticker.fps(30);  // cap at 30fps

// Lag smoothing — prevents big jumps after tab switch
// threshold: if frame takes longer than Xms, cap delta at Yms
gsap.ticker.lagSmoothing(500, 33);

// Get current fps
console.log(gsap.ticker.fps()); // returns current fps

// Global time scale (slow/speed up everything)
gsap.globalTimeline.timeScale(0.5); // half speed universe!

Performance Best Practices

Get the most out of GSAP — smooth 60fps animations at scale

✅ DO — GPU-friendly properties

  • Use x, y instead of left, top
  • Use scale instead of width/height
  • Use opacity for fade effects
  • Use will-change: transform on animated elements
  • Use force3D: true to promote to compositor

❌ AVOID — Layout-triggering properties

  • Animating width, height, padding, margin causes reflow
  • Reading layout values (getBoundingClientRect) mid-animation
  • Animating too many DOM elements without virtualization
  • Creating tweens in scroll/mousemove without throttling
JavaScript — Performance Tips
// 1. Use overwrite to prevent conflicting tweens
gsap.to(".box", {
  x: 200,
  overwrite: "auto" // kills only conflicting properties
});

// 2. Lazy rendering — GSAP renders on next frame by default
// Use immediateRender: false to defer initial state
gsap.from(".box", {
  opacity: 0,
  immediateRender: false // don't apply initial state immediately
});

// 3. Pre-create tweens, then trigger them paused
const fadeIn = gsap.to(".box", { opacity: 1, paused: true });
// Later:
fadeIn.play();

// 4. Force3D — promote element to GPU layer
gsap.to(".box", {
  x: 300,
  force3D: true  // adds translateZ(0) automatically
});

// 5. Use will-change CSS for elements you know will animate
// CSS: .animated { will-change: transform, opacity; }

// 6. Batch DOM reads before animation
const positions = elements.map(el => el.getBoundingClientRect());
elements.forEach((el, i) => {
  gsap.to(el, { x: targetPositions[i].x });
});

// 7. ScrollTrigger.refresh() after layout changes
ScrollTrigger.refresh();

// 8. Kill unused tweens to free memory
gsap.killTweensOf(".old-element");
ScrollTrigger.getAll().forEach(st => st.kill());

import { useRef, useEffect } from "react";
import { gsap } from "gsap";
import { useGSAP } from "@gsap/react";

function AnimatedCard() {
  const container = useRef();

  useGSAP(() => {

    return () => ctx.revert(); // cleanup
  }, { scope: container });
  
  return (
    <div ref={container}>
      <div className="card">
        <h2 className="title">Hello GSAP!</h2>
      </div>
    </div>
  );
}
